# Installing Sui SDK and dApp Kit

This guide will walk you through installing all the necessary packages to build a Sui dApp with React.

## Required Packages

To use the Sui dApp Kit in your React project, you need to install three core packages:

1. **@mysten/dapp-kit** - React components and hooks for Sui dApps
2. **@mysten/sui** - Core Sui TypeScript SDK
3. **@tanstack/react-query** - Required for data fetching (peer dependency)

## Installation Methods

<Tabs>
  <Tab title="npm">
    ```bash npm install @mysten/dapp-kit @mysten/sui @tanstack/react-query ```
  </Tab>

  <Tab title="pnpm">
    ```bash pnpm add @mysten/dapp-kit @mysten/sui @tanstack/react-query ```
  </Tab>

  <Tab title="yarn">
    ```bash yarn add @mysten/dapp-kit @mysten/sui @tanstack/react-query ```
  </Tab>
</Tabs>

## Package Overview

### @mysten/dapp-kit

The dApp Kit provides:

- **React Hooks** for wallet interactions, transactions, and queries
- **UI Components** like `ConnectButton` and `ConnectModal`
- **Provider Components** for managing state
- **Type-safe APIs** for blockchain interactions

**Key exports:**

- `WalletProvider` - Manages wallet state
- `SuiClientProvider` - Manages network connections
- `useCurrentAccount()` - Get connected wallet account
- `useSignAndExecuteTransaction()` - Sign and execute transactions
- `useSuiClientQuery()` - Query blockchain data
- `ConnectButton` - Pre-built wallet connection UI

### @mysten/sui

The core Sui SDK provides:

- **SuiClient** - RPC client for interacting with Sui nodes
- **Transaction** - Build programmable transactions
- **Cryptography utilities** - Key generation and signing
- **Type definitions** - TypeScript types for Sui objects

**Key exports:**

- `SuiClient` - Main client class
- `Transaction` - Transaction builder
- `getFullnodeUrl()` - Get network URLs
- Type definitions for Sui objects, events, and more

### @tanstack/react-query

React Query is a peer dependency that powers the data fetching hooks in dApp Kit:

- Automatic caching and refetching
- Loading and error states
- Optimistic updates
- Request deduplication

## Creating a New Project (Optional)

If you're starting from scratch, you can use the official Sui dApp template:

```bash
npm create @mysten/dapp
```

This CLI tool will:

1. Prompt you to enter a project name
2. Let you choose a template (react-client-dapp or react-e2e-counter)
3. Set up a complete project with all dependencies pre-installed

## Installing dApp Kit CSS (Optional)

If you want to use the pre-built UI components with default styling, import the CSS file in your app:

```typescript
// In your main app file (e.g., main.tsx or App.tsx)
import "@mysten/dapp-kit/dist/index.css";
```

<Note>
  The CSS import is **optional**. You can completely customize the appearance
  using the theming system or by writing your own styles.
</Note>

## Verify Installation

After installation, verify the packages are installed correctly:

```bash
npm list @mysten/dapp-kit @mysten/sui @tanstack/react-query
```

You should see output similar to:

```
your-project@1.0.0 /path/to/your-project
├── @mysten/dapp-kit@0.x.x
├── @mysten/sui@0.x.x
└── @tanstack/react-query@5.x.x
```

## Package Versions

<Info>
  Always use compatible versions of `@mysten/dapp-kit` and `@mysten/sui`. They
  are versioned together and should typically be on the same version number.
</Info>

Check the latest versions:

- [@mysten/dapp-kit on npm](https://www.npmjs.com/package/@mysten/dapp-kit)
- [@mysten/sui on npm](https://www.npmjs.com/package/@mysten/sui)

## TypeScript Configuration

Ensure your `tsconfig.json` includes these compiler options:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  }
}
```
